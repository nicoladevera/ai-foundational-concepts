# LLMs 101

**Learning goal:** Understand how large language models work, what they are good at, where they fail, and why tool use changes their practical value.

---

# What is an LLM?

**LLM = Large Language Model**

At the simplest level, an LLM is a model trained to predict the next token in a sequence. That sounds narrow, but at large scale it produces surprisingly broad capabilities: writing, summarization, coding, translation, question answering, classification, and more.

---

# LLMs as a Machine Learning Application

Large language models are one branch of deep learning.

The rough hierarchy is:
- **Machine learning**: learning patterns from data
- **Deep learning**: machine learning with neural networks
- **Transformers**: a specific neural architecture
- **LLMs**: transformers trained on very large datasets, usually with additional post-training and alignment

## What Makes LLMs Different from Older ML Systems

**Traditional ML**
- Often built for one task
- Usually relies on structured inputs
- Often requires hand-designed features

**LLMs**
- Can handle many tasks through language instructions
- Work well on unstructured text and increasingly multimodal input
- Learn internal representations automatically
- Improve dramatically with scale, post-training, tools, and context

## The Main Ingredients Behind Modern LLMs

- **Self-supervised pretraining** on large text corpora
- **Instruction tuning** so the model follows requests better
- **Preference tuning or reinforcement learning** to improve helpfulness and safety
- **Tool use** so the model can search, retrieve, calculate, execute, or inspect files instead of relying only on memorized patterns

Key idea:
- Modern LLM systems are not just pretrained text predictors. Their practical usefulness comes from the full stack around them.

# How They Work (Simple Version)

## 1. Training Phase

The model sees enormous amounts of text and learns statistical patterns about language, facts, structure, and common sequences.

Example:
- "The cat sat on the ___"
- "def add(a, b): return ___"

## 2. Tokens

LLMs process text as **tokens**, not whole words. A token may be a word, part of a word, punctuation, or whitespace.

Rule of thumb:
- English text averages around 0.75 words per token, but this varies by language and content

Why this matters:
- Context windows are measured in tokens
- API usage is often priced by tokens
- Large codebases or long documents can consume context quickly

## 3. Generation

When you send a prompt, the model:
1. Converts the input into tokens
2. Predicts likely next tokens
3. Repeats until it finishes

This is why LLMs can sound fluent even when they are wrong: fluency and truth are not the same thing.

# Key Concepts

## Context Window

The **context window** is the amount of information the model can process in one interaction.

Analogy:
- It is like the model's working memory for the current request

Important point:
- Context windows vary a lot by model and interface
- A provider may offer different limits in its API, web product, and agent products
- Always check current documentation instead of assuming one number applies everywhere

Why it matters:
- Larger context windows help with long documents, codebases, multi-file tasks, and long conversations
- Bigger context windows do not automatically guarantee better reasoning across all that context

## Parameters

Parameters are the learned weights inside the model.

General rule:
- More parameters usually mean more capacity
- But data quality, architecture, post-training, inference settings, and tool use all matter too

Important caution:
- Exact parameter counts for many frontier models are not publicly disclosed
- Parameter count alone is not a reliable buying guide

## Prompts

The input instructions, questions, examples, or context you provide.

## Temperature

A control for randomness.

- Lower temperature: more stable and focused
- Higher temperature: more varied and creative

## Embeddings

Numerical representations of meaning that make semantic search and retrieval possible.

## Fine-Tuning

Additional training on narrower data to improve behavior, style, or performance for a specific class of tasks.

# Important Limitations

## 1. They Are Not Grounded by Default

An LLM can produce text that sounds knowledgeable without being tied to a source, a tool result, or a verified fact.

That is why grounding matters:
- retrieval
- file search
- web search
- calculators
- code execution
- database access

## 2. Hallucinations

**Hallucination** means the model states something false, invented, or unsupported as if it were true.

Common examples:
- made-up citations
- incorrect statistics
- nonexistent product features
- confident but wrong code explanations

Why it happens:
- The model is optimizing for a plausible next response, not for truth by itself

## 3. Knowledge Cutoffs

Models only know what is available within their training and documented knowledge ranges unless you provide newer information or let them use tools.

Practical implication:
- For current events, current products, current laws, or current prices, you should use retrieval or web-backed workflows

## 4. Context Sensitivity

The same model can give very different answers depending on:
- the system instructions
- the examples shown
- the retrieved documents
- the tools available
- how the task is framed

## 5. Reasoning Is Strong but Imperfect

It is too simplistic to say modern LLMs are "just autocomplete" in the dismissive sense. They can display strong reasoning-like behavior, long-horizon planning, and useful problem solving.

But it is also wrong to treat them as dependable reasoners by default.

Better mental model:
- LLMs are powerful predictive systems that can produce impressive reasoning behavior, but they still need grounding, structure, and verification for important work.

# Why Tool Use Changes Everything

## Standard Chat LLM

You ask a question and get text back.

## Tool-Augmented LLM or Agent

You ask for an outcome and the system can:
- read files
- search a codebase
- call APIs
- browse the web
- run code
- inspect intermediate results
- revise its own work

This changes the practical system from:
- "a model that writes responses"

to:
- "a model that can gather evidence, act, and verify"

That is why agent products often feel far more capable than the same underlying model in a plain chat box.

---

# A Useful Analogy

## LLM = A very capable generalist assistant

- Has seen enormous amounts of text
- Can synthesize and draft quickly
- Often sounds confident
- Still makes mistakes
- Needs the right context and tools to do serious work reliably

## Tool-Augmented Agent = That assistant with access to software

- Can inspect the actual environment
- Can gather missing facts
- Can test and revise outputs
- Can complete multi-step work with less hand-holding

---

> **Key Takeaway**
>
> Large language models are powerful general-purpose prediction systems built on deep learning.
>
> - They are useful because scale, post-training, and tool use make next-token prediction surprisingly general
> - They can show impressive reasoning-like behavior, but they are not reliable by default
> - Their main risks are hallucination, stale knowledge, and context dependence
> - Their practical value rises sharply when you combine them with retrieval, tools, and verification
>
> Understanding these limits is what lets you use LLMs well instead of treating them like oracles.
